# IndieAppStack Affiliate Operations

Use this skill when working on affiliate programs or link placement.

## Program Types

- Affiliate program: content-driven link commission.
- Partner program: agency, integration, referral, co-marketing, or marketplace relationship.
- Referral program: account-level referral link with limited terms.
- Sponsorship: paid placement, newsletter, or dedicated review.
- Direct link: no monetization; still useful for readers.

## Intake Checklist

Record:

- Tool name.
- Program type.
- Status: not applied, applied, approved, rejected, unavailable, paused.
- Application URL.
- Network or platform.
- Commission notes.
- Cookie notes.
- Allowed promotion notes.
- Disclosure requirements.
- Date checked.
- Next follow-up.

## Placement Rules

- Add affiliate links only to articles where the tool is a natural recommendation.
- Use `/go/[slug]` for tracked redirects.
- Use `sponsored nofollow` unless a program requires stricter rel values.
- Keep a visible disclosure near the first monetized CTA.
- Never hide affiliate status in button copy.

## Application Positioning

IndieAppStack is a practical field guide for solo mobile developers choosing tools for monetization, backend, analytics, ASO, launch, screenshots, and developer productivity.

Applications should emphasize:

- Audience fit: solo iOS, Android, React Native, Flutter, and web-to-app builders.
- Editorial standards: verified claims, honest fit guidance, no commission-based rankings.
- Assets: tool pages, comparisons, guides, Stack Finder, newsletter.
- Tracking: redirect click events and privacy-friendly analytics.

## Output

Return:
